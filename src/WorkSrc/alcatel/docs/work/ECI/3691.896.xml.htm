What causes a MARSHAL(REQUEST_MESSAGE_NOT_CONSUMED) exception?

26-Sep-2001 | Product & Version: Orbix 2000, All platforms | Article ID: 3691.896

When I run my application I get the following exception: 

 Unexpected system exception during for_consumers: org.omg.CORBA.MARSHAL:
 minor code: 1230242327  completed: No

I have determined that the minor code 1230242327 corresponds to the minor condition REQUEST_MESSAGE_NOT_CONSUMED. What does this mean?

(Note that this articles also applies to the similar condition
REPLY_MESSAGE_NOT_CONSUMED)

Answer:
This error message indicates a marshalling error, and occurs when the server unmarshalls a Request message from a client. Once all the components of the message have been unmarshalled (headers, parameters, operation contexts etc) the stream checks to make sure no bytes are left over - that is, it checks to ensure that the message has been completely consumed.

The minor code REQUEST_MESSAGE_NOT_CONSUMED indicates trailing bytes are left over. At the time of writing, a discussion was in progress within the OMG regarding this issue, and indications were that it is likely to be deemed illegal to have trailing bytes in a GIOP message which are not part of the marshalled data.

There are a number of possible reasons that there may be trailing "left over" bytes . For example, the client could have set the wrong message length (and then transmit trailing junk bytes), the client could have the alignment wrong (so the server unmarshalls it with different alignment to the client), a sequence length is wrong, or perhaps even the IDL is different between client/server leading to different parameters. Some older ORB implementations even added their own data to GIOP messages by piggy-backing trailing data onto the end of messages before later implementations began to consider the condition illegal.

To help determine the cause you will need to use a GIOP protocol debugging tool such as the GIOP Snoop plug-in that comes with Orbix 2000 to examine the complete message. For details on how to run GIOP Snoop please read giop_snoop_README.txt which is included in your installation.

Note that as of Orbix2000 version 2.0 there is a configuration setting to relax the check for trailing bytes in a message. The setting

 policies:giop:interop_policy:ignore_message_not_consumed

if TRUE will not result in the exception condition described above. Default value is FALSE. Any trailing bytes will simply be ignored.
